## 2024-04-09 - Avoid Nested Interactive Elements for Accessibility
**Learning:** Placing a `<button>` (like a delete action) inside another `<button>` (like a clickable tile or card) is invalid HTML and severely breaks screen reader navigation, as assistive technologies expect buttons to be discrete, focusable leaf nodes. Attempting to fix this by hiding the inner element from screen readers (`aria-hidden="true"`) and moving key handlers to the parent degrades the user experience for users who navigate via swipe gestures or explicit focus.
**Action:** Always structure interactive cards/tiles using a non-interactive container (like a `<div>` or `<li>`) and place the individual interactive elements (e.g., the main action `<button>` and the secondary action `<button>`) as siblings within it. Add explicit ARIA labels to ensure context is preserved for each action.
