
## 2025-02-14 - Exposing Range Slider Values explicitly
**Learning:** Range sliders (`<input type="range">`) provide good macro-interaction for picking values within a range, but without accompanying text displaying the exact current value, they introduce guesswork and poor accessibility, particularly when precise configuration is desired (e.g. animation speed). Additionally, since these values often come from un-initialized settings or local storage states, relying on standard JS logic `||` can crash if the value is meant to be `0` but evaluated falsy, causing unexpected fallback values or `.toFixed()` TypeErrors.
**Action:** Always pair `<input type="range">` elements with a visual `<span className="settings-panel__hint">` indicator of the explicit current numeric value. Always use robust type casting `Number(value ?? fallback).toFixed(decimal)` and the nullish coalescing operator `??` instead of `||` to gracefully handle `0` or explicitly missing values to avoid crashes.
