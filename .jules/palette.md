## 2025-02-28 - Nested Delete Button Accessibility
**Learning:** In a heavily customized grid or list (like the App's Settings SkinGrid or ShelfGrid), interactive elements like `<button>` elements shouldn't be nested inside other `<button>`s, but placing non-button "targets" (like a `<span>` shaped like an X) inside a clickable parent creates keyboard accessibility issues because the inner element cannot receive separate tab focus easily.
**Action:** The standard UI pattern in this codebase for nested item deletion targets is to add `onKeyDown` handlers catching `Delete` and `Backspace` on the parent container (e.g. `.shelf-item` or `.skin-tile`), combined with adding `:focus-visible .child-delete-btn` and `:focus-within .child-delete-btn` CSS rules. This ensures keyboard users discover and can trigger the action efficiently without HTML validation or complex tabbing order issues.
