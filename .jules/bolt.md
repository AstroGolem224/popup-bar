## 2024-03-23 - Memoizing Reorderable Grid Items
**Learning:** High-frequency state changes during drag-and-drop operations (like `mousemove` dragging) trigger expensive re-renders across the entire `ShelfGrid` component if items are not memoized correctly. In order to effectively use `React.memo` for the individual `ShelfItem` components, they must not receive inline objects (like a dynamically generated `style` object) or unstable callback props (like `onReorderMouseDown` or `onDelete` recreating on every render).
**Action:** Always extract layout coordinates to primitive component props (e.g. `positionX` and `positionY` instead of inline `style` objects) and wrap the component in `React.memo()`. Pair this with the latest-value ref pattern (`useRef` + `useEffect` synchronizing state) inside custom hooks or parent components to ensure callbacks remain stable and do not break the memoization.
