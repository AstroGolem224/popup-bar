## 2025-03-27 - Inline style objects and unmemoized callbacks breaking React list performance
**Learning:** In a heavily interactive component like `ShelfGrid`, passing inline style objects (`style={{ left: x, top: y }}`) and unmemoized callbacks directly to child list items breaks `React.memo()`. This forces the entire list of `ShelfItem` components to re-render on every single pixel movement during drag-and-drop, causing significant frame rate drops. In `useItemReorder`, closing over changing React state (`items`, `dragPositions`) inside a `useCallback` also forces the handler to be recreated constantly, further breaking memoization of child items relying on that handler.
**Action:** Always decompose inline style objects into primitive props (like `positionX` and `positionY`) for memoized list children so they can be shallowly compared. To provide stable callbacks that need access to frequently updating state, use the 'latest-value ref pattern' (`latestStateRef.current = { ...state };`) inside `useRef` and access it within an empty-dependency `useCallback`.
