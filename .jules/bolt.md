## 2024-05-18 - Drag and Drop Re-renders
**Learning:** Frequent state updates for positioning coordinates during a local drag-and-drop interaction cause high-frequency unoptimized re-renders across the entire shelf grid. `ShelfItem` components previously re-rendered every time any single item was moved. React.memo requires stable props: avoiding inline style objects and stabilizing parent callbacks (via empty-dependency useCallback and latest-value ref patterns) prevents this.
**Action:** When working with local positioning state (like `useItemReorder`), always enforce strict memoization boundaries via `React.memo` on child nodes (like `ShelfItem`) and stabilize inline functions in their parents using `useCallback` along with refs for state access. Pass primitive layout variables rather than structural objects.
