## 2024-03-17 - Stabilizing React Trees with Tauri Window Events
**Learning:** High-frequency Tauri commands like `hotzone:enter`/`hotzone:leave` cause root-level app visibility state to toggle often. Because standard context/hooks propagate state deeply, large lists (like `ShelfGrid` / `ShelfItem`) can re-render repeatedly just because the parent bar's animation or visibility state changed.
**Action:** When passing callbacks to deeply nested list elements, always use `useCallback`. Do not use reactive `useStore((s) => s.items)` arrays in dependency arrays for event handlers like drag-and-drop; instead, dynamically read the state using `store.getState()` right when the handler fires. Wrap list/item components in `React.memo` to ensure they only update if their exact properties change.
