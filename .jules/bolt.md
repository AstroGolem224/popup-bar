## 2024-05-18 - [Preventing Unnecessary Re-renders in Drag and Drop Hooks]
**Learning:** In a drag-and-drop React implementation using Zustand, subscribing directly to the `items` array inside the `onDrop` callback hook (like `useShelfStore((state) => state.items)`) forces the entire drag-and-drop context (and thereby the components consuming it) to re-render whenever *any* item changes or moves. This causes severe performance degradation and deep re-render trees during high-frequency interaction.
**Action:** When a callback only needs to *read* state to process an event (like calculating a new drop index) but doesn't need to trigger a render when that state changes, always read the state imperatively using Zustand's `store.getState()` instead of a reactive hook subscription. Combine this with `React.memo()` for the list children.
