## 2024-03-22 - Stabilizing React state dependencies during high-frequency events
**Learning:** In components with high-frequency interactions (like `useItemReorder` firing `mousemove` multiple times per second), passing callbacks that close over React state directly to memoized children (like `ShelfItem`) breaks memoization entirely. Every update to `dragPositions` causes a new reference of the array/object to be created, and if the event handler `useCallback` lists it in the dependency array, the handler reference changes. This results in every `ShelfItem` re-rendering on every mouse move, leading to extreme input latency and a choppy experience.
**Action:** When a callback needs the latest React state but must maintain referential stability for memoized children, use the "latest-value ref pattern". Store the state dependencies in a `useRef` (updating it via `useEffect` without dependencies, or directly before accessing) and omit the state from the `useCallback` dependency array. This guarantees `ShelfItem` components only re-render when their *own* props (like `positionX` or `isDragging`) actually change.
